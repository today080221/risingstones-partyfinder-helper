# Roadmap

## 已完成第一版

### 可配置更新源

目标：让便携包用户可以从 GitHub、Gitee 等发布源检查并获取新版本。

当前能力：

- 支持 GitHub Releases。
- 支持 Gitee Releases。
- 固定维护 GitHub 主仓库和 Gitee 国内镜像两个下载节点。
- 支持加载时 GeoIP 推荐下载节点：中国大陆或检测失败默认 Gitee，海外推荐 GitHub。
- 支持绿/黄/红更新状态灯。
- 默认只“检查并提示”，不静默替换本地文件。
- 下载更新包前展示版本号、发布时间、发布说明和来源地址。
- 不执行未经用户确认的更新包。
- 保留当前版本、最新版本、发布页和附件下载入口。

## 下一优先级

- 增加 release manifest URL 模式，方便自建镜像源。
- 增加 zip 校验文件展示和校验。
- 评估半自动替换流程，但必须保留用户确认。

## 后续增强

- 更多时间文本解析样例。
- 更多职业和队伍构成异常字段兼容。
- Release 包签名或校验文件。
- 用户自定义筛选预设导入导出。
- 更多平台便携包，例如 macOS/Linux。
