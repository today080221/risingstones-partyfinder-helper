# NGA 本地登录态聚合

## Goal

在用户明确授权、手动登录 NGA 后，整理用户自己账号可正常浏览的公开招募帖内容，生成样本分析报告，并用已确认黑话落地第一版本地 Parser。

## Confirmed Requirements

- 主体验面向 Tauri 桌面版：用户在应用内 NGA WebView 中自行登录。
- 支持“保持登录状态”，默认关闭；首次开启前必须弹出风险提示并由用户确认。
- 保持登录只使用 WebView 普通本地浏览器数据目录，不读取、不导出、不上传 Cookie、token、localStorage、sessionStorage 或账号凭据。
- 允许用户随时清除本机 NGA 登录状态；清理范围只限本应用 NGA WebView profile，不影响系统浏览器。
- 所有采集动作必须在 UI 中可见、可取消、有限频，并由用户明确触发。
- 采集范围只限用户当前账号可正常浏览的招募帖内容。
- Parser 只能吸收真实样本中出现过、或明显影响 FF14/NGA 招募解析的术语；低置信黑话必须保留为 warning/tag 或未识别，不能强行归类。

## Safety Boundaries

- 不向用户索要 NGA 账号、密码或任何认证信息。
- 不在聊天、日志、配置文件、数据库或错误报告中输出账号凭据或会话数据。
- 不实现自动登录。
- 不绕过验证码、风控、访问权限、站点限制或反自动化机制。
- 不模拟攻击、不扫描、不爆破、不探测接口。
- 不做后台不可见采集。
- 不保存私信、账号安全信息、用户个人主页详情或与招募解析无关的个人资料。

## Sample Fields

NGA 样本只允许包含 Parser 所需白名单字段：

- `title`
- `body`
- `url`
- `author`
- `publishedAt`
- `updatedAt`
- `forumId`
- `topicId`

本地 cache 允许额外保存仅用于增量复核的元数据：`lastCheckedAt`、`lastSeenAt`、`detailFetchedAt`、`contentHash`、`closedAt`、`sourceBoardUrl`。这些字段不包含网页会话数据或账号凭据，只用于判断是否需要重新读取、是否已关闭以及来自哪个招募板。

- 日服招募板块：`https://bbs.nga.cn/thread.php?stid=42005319`

## Parser Training Workflow

- 先采集真实样本，并生成报告：常见标题结构、正文结构、字段出现率、字段缺失率、高频副本简称、高频进度表达、高频职业/位置缩写、高频时间表达。
- 将无法确定的黑话或歧义表达输出为“待用户确认问题清单”。
- 用户确认后，把确认结果写入词典、规则或低置信 warning/tag。
- 本轮 Parser v1 已按真实样本分流：队伍招人、玩家求职、补人/替班/临时、固定队/长期队、已关闭/已招满、低置信未识别。
- 灰机 wiki HTML 仅作为本地辅助来源，使用脚本提取正文中与 FF14/NGA 招募解析相关的黑话、缩写、别称和解释；不把整份 HTML 或无关梗塞进 Parser。

## Acceptance Criteria

- NGA 面板默认只展示普通用户需要的地区选择、桌面可用摘要、已保存招募数和必要进度；网页会话、URL、采集参数和手动采集按钮收在“高级设置”。
- 本机路径、样本保存位置、样本分析报告和待确认问题只在开发诊断折叠区展示；生产普通用户默认不可见。
- NGA 地区支持国服、日服、欧区、大洋洲区、美区多选；默认仅选国服。
- 聚合采集按已选 NGA 地区顺序进行，总读取上限是本次 NGA 聚合采集预算，不是每个地区各自预算。
- NGA 聚合默认按最近 14 天活跃主题扫描；列表页能解析到活跃时间时，旧主题默认不再打开正文。高级设置可调整天数，`0` 表示不限。
- 已保存 NGA 招募在启动或前端刷新后优先进入当前列表；超过默认 12 小时未复核、缺正文或状态不确定的招募进入待刷新队列，但仍会先显示给用户。
- 样本采集字段只包含白名单字段。
- 采集流程可取消，请求间隔和最大采集数量生效。
- 异常停止后 UI 状态恢复，已有样本仍可用于分析。
- 分析报告可生成，并包含待用户确认问题清单。
- Parser v1 输出结构化字段、字段置信度、证据片段、warning 和 tag；低置信内容不强行进入核心字段。
- 现有石之家官方数据源卡片和筛选不受影响。
- Playwright e2e 覆盖 NGA 面板首屏渲染、保持登录默认关闭、风险提示取消/确认分支。

## Implemented Scope

- Tauri 桌面版提供用户可见的 NGA 网页窗口，默认按公开可见页面读取；本机网页会话能力只作为高级兜底。
- “保持本机网页会话”默认关闭，首次开启前弹出提示，确认后才保存该偏好。
- 保持本机网页会话使用本应用专属网页数据目录；关闭时使用临时会话。
- 支持清除本机 NGA 网页状态，清理范围限定在本应用专属目录。
- 支持用户显式触发当前可见页面样本采集，采集过程显示进度、可停止、受请求间隔和最大采集数量限制。
- 支持“采集详情正文”：从当前列表页识别 `read.php?tid=...` 帖子后，按请求间隔在同一个可见 NGA 网页窗口中逐个打开帖子详情页补齐正文，不做后台不可见采集；正文按“首楼 + 楼主有效补充”拼接，过滤顶帖和纯占位回复。
- 列表页和详情页会提取 `updatedAt` 活跃时间；聚合读取默认使用“最近活跃天数”过滤需要打开正文的帖子，并在连续旧列表页后停止继续翻页。
- 本地 cache 增量复核：聚合读取会先扫描新版面新主题，再按 12 小时默认复核间隔静默更新旧主题；未变化主题只刷新复核时间，不重复打开正文。
- 支持本地样本池自动保存到应用数据目录，样本去重累积，上限 1500 条；多次“采集当前页”是 additive，不覆盖上一批样本。
- 支持“补齐已存正文”：当已有样本只有标题和 URL 时，可以直接按已存 `read.php?tid=...` 链接在可见 NGA WebView 中逐个打开详情页补正文；同一主题的新正文样本会覆盖旧的标题-only 样本。
- NGA 起始页面提供国服、日服、欧区、大洋洲区、美区招募板快捷入口。
- 无正文样本进入本地样本池前必须具备招募信号；明显广告、公告、游戏资讯、首页推荐标题会被丢弃。带正文样本仍保留给分析报告和低置信判断。
- 疑似已招满、已关闭、广告、公告、非 FF14 招募内容会保留在样本池和分析报告中，但平衡档默认不进入招募卡片；未识别档可查看全部样本。
- 支持“登录后采集”一次性待命：用户触发后打开 NGA WebView，最多等待 15 分钟，按请求间隔检查当前可见 NGA 页面；识别到招募样本后立即停止待命并生成报告。
- 支持右上角“聚合检索”统一入口：按左侧多选来源处理石之家与 NGA；石之家未选副本时只跳过石之家并提示，不阻断 NGA。
- 结果来源从单选改为多选，默认全选石之家和 NGA，并保留后续接入百度贴吧等来源的扩展点。
- 侧栏二次精简：数据来源置顶，随后是石之家/NGA 共用的数据范围和通用招募筛选；标签/类型改为本地筛选，统一匹配石之家标签与 NGA Parser tag/要求。
- NGA 普通面板已精简：地区多选、桌面可用摘要、已保存招募数和进度常驻；“高级设置”内包含本机网页会话、招募板 URL、翻页间隔、最多读取、正文采集、筛选档位和手动采集命令。
- 默认 NGA 采集参数调整为翻页间隔 1 秒、本次最多读取 500 条、同时打开帖子正文；高级设置允许最短 0.5 秒，Tauri 侧同样将最小请求间隔 clamp 到 500ms。
- 高级设置中“本次最多读取”已明确为“本次最多扫描”，表示本轮最多查看多少个列表主题；最近活跃过滤命中后才会打开正文并保留到本地样本池。
- 开发诊断只在 Vite dev 或本地调试开关下出现，默认折叠；包含路径、样本保存位置、样本分析报告和待确认问题。
- NGA 卡片标题统一使用 Parser 识别出的副本名，未识别时显示“未识别副本”；原帖标题只进入副标题，并清洗作者等级、威望、注册日期等 NGA 元信息。
- 聚合读取在打开或切换地区后会轮询当前可见页面；如果遇到 `misc/adpage_insert_2.html?...` 继续浏览页，会优先尝试点击页面上的继续按钮，失败时再提示用户处理，不再直接报“不支持页面”。
- 聚合读取窗口默认以最小化方式打开，手动“打开 NGA”仍正常显示；用户需要处理页面提示时可手动恢复窗口。
- NGA WebView 支持受控新窗口，用于第三方登录等页面弹出的授权窗口；这些窗口仍在本机 WebView 中，不读取认证数据。
- 样本进入本地 UI 状态、本机样本 JSON 和分析报告，不包含账号凭据、会话数据或无关个人信息。
- 官方和 NGA 招募统一进入结果列表，卡片通过来源徽标区分，详情按钮统一为“查看详情”。
- 结果区顶部提供 cache 友好的统一读取状态条，集中展示所选来源、本地已保存、当前命中、待刷新和最近复核时间；聚合读取时显示“新增 / 待刷新 / 已复核”的轻量进度。
- 结果列表使用虚拟列表渲染大量可变高度卡片；NGA 新增/更新数据按稳定 ID 热插入，已有卡原地更新并短暂高亮，插入或隐藏时尽量补偿滚动位置。
- Parser v1 已接入 NGA 招募卡片、详情面板和本地筛选：卡片展示解析状态、职业/位置、tag、warning；详情面板展示结构化字段、置信度和证据片段。
- 新增 Playwright e2e 配置和 `npm run test:e2e`，默认使用 Playwright 自带 Chromium/headless shell；必要时可通过 `PLAYWRIGHT_CHANNEL=chrome` 改用本机 Chrome。
- NGA 详情面板改为可读招募卡：按“当前队伍构成/玩家可用位置、队伍详情、招募要求、攻略说明、原文参考”组织；来源显示为“石之家”或“NGA”，避免用“官方”代指石之家。
- 时间展示统一使用 24 小时制区间，去重重复时间和被包含子区间，单独展示 `10-12h`、`2-3小时` 等每日强度字段；NGA 和石之家卡片共用同一展示规则。
- 当前队伍构成支持职业图标；同一槽位多职业可切时以叠层卡片展示，主职业在顶层，鼠标悬停或键盘聚焦时展开候选职业。

## Current Parser Boundary

当前实现为 Parser v1：规则 + 词典 + 启发式分流，不读取认证数据，不依赖在线服务。Parser 输出包括 `parsedFields`、`parseConfidence`、`parseEvidence`、`parseTags` 和 `parseWarnings`。

- 已支持：队伍招人、玩家求职、补人/替班/临时、固定队/长期队、已关闭/已招满、低置信未识别；石之家和 NGA 共用本地标签/类型筛选口径。
- 已支持：副本简称、大区简称、队伍类型/过本目标标签、进度、攻略文本、活动时间、招募位置、求职可用位置、职业、排除职业/位置、9 人轮换/第九人、联系方式方法、logs/插件/代打/队伍社交属性等要求标签。
- `D1-D4` 在招募语境下默认按招募位置高置信处理；只有附近出现“机制位/攻略位/站位/分摊/散开”等上下文时才降级为需人工确认。
- `绿玩/红玩/轮椅/科技/装甲车` 只作为 warning/tag 和 requirements，不进入副本、进度或职业强判断；正向“需要/允许/可用轮椅/科技/非绿玩”等表达标为可能涉及第三方插件等风险。
- `红玩绿玩均可`、`绿玩洁癖`、`没挂水平下降` 等表达按中性或反依赖标签处理；裸 `ACT` 不单独作为第三方风险，`ACT` 只在查输出、logs 记录、个人复盘语境中降级为 `ACT/logs 记录复盘`。
- `禁止/谢绝/婉拒插件/第三方/外挂/宝宝椅`、`非装甲车`、`装甲车过本看不到我`、`不要代打/工作室记录` 等反提示词解析为 `纯净队/禁第三方` 或 `拒绝装甲车/代打记录`，不误标为队伍使用第三方工具或代打风险。
- `婉拒无绘图轮椅不会打本` 等表达解析为 `拒绝绘图轮椅依赖`，不等价于纯净队，也不反向标记为队伍使用轮椅。
- `时间轴` 单独出现时按官方日程、副本时间线或减伤轴文本处理，不标第三方风险；只有和 `ACT/TTS/轮椅/播报/Triggernometry/PostNamazu` 等工具词相邻时才标为 `ACT 时间轴/TTS 辅助`。
- `已有/现有阵容/队伍配置/队内已有` 等上下文中的职业和位置按已占坑处理，不进入空缺；标题或正文里的 `缺/招/补/需求职业/7=1 D1` 等明确招募上下文才进入 `positions/jobs`。
- `队内配置/阵容/已有` 后直接连写职业时，按国服默认 `MT/ST/H1/H2/D1/D2/D3/D4` 顺序解析；若附近有 `7=1 D4`、`6=2 H2 D1` 等空缺信息，会先跳过空缺槽位再填已有职业，避免把后续职业错位。
- `武士/蛇(d1)`、`舞者/机工(d3)`、`画家/黑魔(d4)` 等 `职业/可切职业(位置)` 写法按括号中的位置落入当前队伍构成，括号后的文字不再被错当成该位置职业。
- `招募D2(侍、龙、忍、武僧)` 等需求职业候选组会完整进入招募职业字段；`侍/黑爷/黑黑的爷/画家` 等样本中出现的别称已纳入职业词典。
- 明显部队/公会社交帖如果没有副本、进度或位置证据，进入噪声流；标题和正文都被清成 `编辑+数字` 或纯数字占位的帖子进入已关闭流。
- `全妹队/女生限定`、`女生优先` 作为队伍属性/要求标签展示，使用独立标签色。
- `contact` 只输出联系方式方法标签；证据片段中的长数字串会脱敏，避免 Parser 结构化输出额外扩散个人联系方式细节。
- `ed` 只在独立 token 中映射光暗未来绝境战，`edT` 等连续 token 不拆分误判。
- `tndd` 按 T/N 和 DD 的职能分组攻略表达进入 strategy；`mmw/猫猫窝` 映射猫猫窝攻略组文本。
- `M1S-M12S` 等 M 系零式简称映射为 `阿卡迪亚登天斗技场 MxS` 或范围文本。

## 2026-05-09 Parser/UI Pass

- 新增 `scripts/validate-nga-parser.ts` 专项回测脚本，默认读取本机 `nga-samples.json`，校验样本池仍为 396 条且全部具备正文，再对用户标注过的高难 case 做结构化断言。
- Parser 对 `7=1 D4`、`6=2 H2 D1`、`5=3 h1 h2 d4`、`4=4 (MT ST D3 D4)` 等标题或正文强约束采用优先口径；若正文没有更强的招募人数/空位说明，标题中的 `几=几 + 位置` 可作为高置信招募位置。
- 职业排布进一步区分“当前阵容”和“招募需求”：`已有/现有/队内配置/职业构成/阵容` 后的职业默认落入当前队伍构成，`招募/需求/缺/空` 后的职业和位置才进入需求字段。
- 当前队伍构成支持更多写法：连写职业、括号位置、`2T/双奶/2近` 计数职能、`+` 连接、`或/or/、/，/；/_/~` 等常见分隔符、`D1/2` 与 `MT/ST` 这类同职能可切槽位。
- 同职能可切会合并为组合槽位：例如 `MT/ST`、`D1/D2`、`H1/H2` 表示一个人可打两个位置；若原文明确分别招 `H1 H2`，则保持拆开的两个空位，不合并。
- 需求职业候选保持“或”的关系展示：`招募D2(侍、龙、忍、武僧)`、`D4 黑魔/画家/赤魔` 会保留候选组，不误读成多个同时需求的职业。

## Confirmed Parser Notes

- “求职”表示单个玩家找队伍，希望队长捞人；后续 Parser 应按求职流处理，而不是队长招人流。
- “绝区零”“游戏评测”“版务公告”不是 FF14 副本招募黑话，应视为噪音或非招募内容。
- “艾欧泽亚”是 FF14 世界/大陆名称，不能直接当副本名。
- “绝妖星乱舞”在用户给出的样例中指向“妖星乱舞绝境战”；“绝”通常代表绝境战，但具体副本映射仍要以样本和用户确认后的词典为准。
- “社畜队”通常表示队员多为下班后可活动的人，时间倾向晚间。
- “7=1”表示 8 人满编小队中已有 7 人、缺 1 人；“7=1 D1/2”表示缺 D1 或 D2，D1/D2 是近战位。

- `N/奶/绿/绿职` 在职位语境中指治疗职能；但 `绿玩` 不是治疗，指不开 ACT/Dalamud 等插件或只使用有限功能的玩家，语义存在圈内争议。
- `红玩/非绿玩` 泛指使用第三方插件、脚本或外部辅助较多的玩家；`红玩绿玩均可` 是中性插件生态标签，不等同于风险；`轮椅` 通常指 ACT 时间轴/TTS 机制播报；`科技` 通常指 Dalamud、外挂、宝宝椅或其他可提供额外优势的第三方软件；`装甲车` 指代打/工作室式打手带老板过本。

- `非绝枪MT`、`非蛇D1`、`非龙近战` 表示缺某位置但排除某职业；`非D4` 需要注意它可能表示机制/攻略站位不是 D4，而不必然等同于非远程法系职业。

- `logs` 颜色、`灰/绿/蓝/紫/橙/粉/金`、`med紫`、`非灰0` 属于招募要求或风险标签，不是副本进度；`med` 指 median，中位数；`非灰0` 指不要 FFLogs ranking 为 0 或明显被拖过本的记录。
