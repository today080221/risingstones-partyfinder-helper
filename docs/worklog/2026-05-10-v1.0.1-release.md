# v1.0.1 补丁版发布准备

## 开始状态

- 当前分支：`main`。
- 开始前 `git status --short --branch`：干净，`main` 与 `origin/main` 同步。
- 已执行 `git fetch origin` 与 `git fetch gitee`。
- `HEAD`、`origin/main`、`gitee/main` 均为 `9a3ba82b9ca09cac2e43f1084561bb3e096a5afe`。
- `v1.0.1` 本地 tag 不存在。

## 目标

- 发布 `v1.0.1`，让已拿到单源 `v1.0.0` 的客户端可以通过版本号检测到补丁更新。
- 发布包必须同时内置 GitHub 与 Gitee 更新源。
- 本轮只做 release prep、版本同步、验证、打包与发布，不修改功能代码。

## 计划

- 同步 npm、Cargo、Tauri 版本号为 `1.0.1`。
- 更新变更日志和发布文档中的当前包名示例。
- 运行测试、构建、Tauri 桌面便携打包和双源 manifest 检查。
- 提交版本同步，创建并推送 `v1.0.1` tag。
- 创建 GitHub Release，并尽可能发布 Gitee Release。

## 风险

- 同版本 SHA 变化不会触发旧客户端更新，因此必须使用 `v1.0.1` 作为补丁版本。
- Gitee Release API 需要本机有效 `GITEE_ACCESS_TOKEN`；不得在仓库或聊天中记录 token。

- `package.json`、`package-lock.json` 版本同步为 `1.0.1`。
- `src-tauri/Cargo.toml`、`src-tauri/Cargo.lock`、`src-tauri/tauri.conf.json` 版本同步为 `1.0.1`。
- `CHANGELOG.md` 新增 `1.0.1` 补丁说明。
- README 和 release 文档中的当前包名示例更新为 `v1.0.1`。
