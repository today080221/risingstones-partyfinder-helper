# v0.1.5 一键更新与发布加固工作记录

## Start

- 开始：用户反馈便携版里“拉取全量”按钮呈黑色，且更新检查只给发布页/附件跳转，不能直接一键更新。
- 关联问题：用户本机普通 PowerShell 构建桌面便携包时 Cargo/VS 环境不稳定；Gitee Release 上传资产时出现 TLS socket 中断。
- 目标：
  - 修正主按钮禁用和可点击状态的视觉表达。
  - 在 Node 便携版和 Tauri 桌面便携版中加入用户确认式一键更新。
  - 保持账号、Cookie、Token、国内镜像真实地址不进入公开源码或文档。
  - 加固本机发布脚本，降低“构建环境没加载”和“Gitee 上传中断”的人工处理成本。

## Requirement Alignment

- 一键更新不是静默更新：必须由用户手动点击并确认。
- 更新范围限定当前便携包解压目录；目录内必须存在 `release-manifest.json` 才允许覆盖。
- 更新包只接受本项目 Release 中的 zip：
  - 桌面版只安装 `desktop-win-x64-portable` 包。
  - Node 便携版只安装 `win-x64` 包，且不安装桌面包。
- 开发模式、源码目录、缺少 manifest 的目录不执行覆盖更新。
- 发布页链接只作为备用信息；主要路径改为“选中更新包 -> 一键更新”。
- 国内镜像配置仍只通过发布机环境变量、未提交本地配置或 release manifest 注入。

## Implementation

- 前端：
  - `primary-button` 改为更明亮的绿色状态，禁用态改为浅灰绿，避免看起来像黑色异常按钮。
  - 更新卡片根据当前运行形态自动选择合适 zip，展示“一键更新”按钮。
  - 用户确认后调用统一 `installUpdate(...)` API；执行中显示下载状态，失败时显示错误。
- Node 便携包：
  - 新增 `POST /api/update/install`。
  - 服务端校验包名、下载源、运行形态和 `release-manifest.json`。
  - 下载 zip 后生成临时 PowerShell 脚本，等待当前进程退出，解压覆盖并重启 EXE。
  - Gitee Release 附件解析兼容 `attach_files` / `download_url`。
- Tauri 桌面版：
  - 新增 `risingstones_install_update` 命令。
  - Rust 层执行同等的包名/来源/manifest 校验、下载、生成覆盖脚本、退出并重启。
- 发布脚本：
  - `scripts/run-tauri-build.mjs` 自动处理 `~/.cargo/bin` 和 VS C++ 环境。
  - 桌面便携版 `--no-bundle` 路径改为临时 `.cmd` 内加载 `VsDevCmd.bat` 后直接执行 `cargo build --release`，避开 Tauri CLI 转发 Cargo 时丢失 Windows SDK 路径的问题。
  - 针对 npm 启动链可能只暴露 MSVC 路径、缺少 Windows SDK 的情况，构建前会主动扫描 Windows Kits，生成临时 `link-wrapper.cmd` 和 `build-env.cmd`，补齐 `LIB`、`LIBPATH`、`INCLUDE`。
  - `scripts/publish-gitee-release.mjs` 同步上传 `.sha256`，并对 socket 中断、5xx、429 做重试；重复资产保留远端文件。
- 版本：
  - 升级为 `0.1.5`，用于发布一键更新和构建脚本加固。

## End

- 本轮已完成主按钮视觉修正和用户确认式一键更新。
- 由于 v0.1.4 客户端本身没有一键更新能力，从 v0.1.4 升到 v0.1.5 仍需要用户手动下载一次；安装 v0.1.5 之后，后续版本即可在客户端内一键更新。
- 若要让国内镜像下载包首次进入就能使用国内镜像更新，需要发布机在构建 v0.1.5 zip 前设置 `RISINGSTONES_UPDATE_GITEE_REPO` 或本机 `config/release.local.json`。
- GitHub Release 已发布；国内镜像 Release 资产上传仍需在发布机本地设置有效 `GITEE_ACCESS_TOKEN` 后执行 `npm run release:gitee`，不得使用聊天中曾暴露过的旧令牌。
- 未签名 EXE 仍可能被浏览器或 Windows SmartScreen 提示未知发布者；一键更新不替代代码签名。
