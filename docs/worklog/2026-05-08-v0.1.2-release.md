# v0.1.2 Release 工作记录

## Start

- 开始：用户要求先发布一个新 Release，然后准备后续签名工作。
- 背景：`v0.1.1` 已经发布，但最新主分支新增了 EXE 便携版入口，用户需要下载 zip、解压、双击 EXE 的开箱即用包。
- 目标：发布 `v0.1.2`，包含 `RisingStones-PartyFinder.exe` 便携入口，并补齐签名准备文档。

## Requirement Alignment

- `v0.1.2` 作为 EXE 便携入口首个公开 Release。
- GitHub Release 由 tag workflow 自动构建和上传 zip。
- `main` 和版本标签仍需同步推送 GitHub `origin` 和国内镜像 `gitee`。
- 国内镜像真实仓库地址和 Gitee token 仍不写入公开源码或文档。
- 本轮先发布未签名包；签名证书、密钥管理和 CI 签名作为下一阶段准备。

## Implementation

- 版本从 `0.1.1` bump 到 `0.1.2`。
- 更新 `CHANGELOG.md`，记录 EXE 便携入口、Tauri 原型和签名准备说明。
- 新增 `docs/release/code-signing.md`，记录 Windows 代码签名准备事项。
- `scripts/build-portable.mjs` 增加 zip SHA256 生成，产物为 `.zip.sha256`。
- GitHub Release workflow 改为同时上传 zip 和 `.sha256`。

## Verification

- `npm run release:check`：通过，包含 12 项单测、前端构建、后端打包和 EXE 便携包生成。
- EXE 启动探针：
  - 设置 `PORT=8902`、`AUTO_OPEN_BROWSER=false` 后启动 `RisingStones-PartyFinder.exe`。
  - `/api/health` 返回 `ok=true`。
  - `/api/version` 返回 `version=0.1.2`、`portable=true`。
