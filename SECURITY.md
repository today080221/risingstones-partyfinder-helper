# 安全策略

## 支持范围

当前维护分支：

- `main`

发布包以 GitHub Releases 中的最新版本为准。

## 安全边界

本项目明确不做以下事情：

- 不保存石之家账号、密码、Cookie、Token 或联系信息。
- 不读取官方站点的 HttpOnly Cookie。
- 不在本地工具中直接代替用户响应招募。
- 不自动提交响应招募请求。
- 不绕过官方登录、角色绑定或官方风控。

本项目允许的账号相关辅助只有：

- 打开官方详情页，让用户在官方页面内自行操作。
- Tampermonkey 脚本在官方域名下运行，并且必须由用户手动确认。

## 报告安全问题

如果你发现安全问题，请不要在公开 Issue 中贴出账号、Cookie、Token 或可复现滥用细节。

推荐做法：

1. 在 GitHub Issue 中提交不含敏感信息的摘要，并标记为安全问题。
2. 如果仓库配置了 Security Advisory，请优先使用 GitHub 私有安全通告。
3. 等维护者确认后，再补充最小复现信息。

## 敏感信息处理

请不要把以下内容提交到仓库或 Issue：

- `.env` 文件。
- `config/release.local.json`。
- 浏览器 Cookie。
- 请求 Header 中的鉴权字段。
- Gitee/GitHub 个人访问令牌。
- 个人联系方式。
- 游戏账号、角色绑定或私聊截图中的隐私信息。

如果令牌曾经出现在聊天、Issue、日志或命令历史中，请立即撤销并重新生成；不要继续使用已经暴露过的令牌。
