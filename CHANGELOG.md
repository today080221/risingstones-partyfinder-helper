# 变更日志

本项目遵循“面向用户说明变化”的轻量变更日志。正式 Release 建议使用 `vX.Y.Z` 标签。

## 1.0.2

- 标签/类型筛选多选默认改为“全部命中”，更适合同时筛选多个队伍属性。
- 在标签/类型标题行新增“任一命中”小开关，需要旧的任一标签命中行为时可点亮切换。
- 旧本地筛选状态自动迁移为默认全部命中，不影响既有标签选择。

## 1.0.1

- 重新发布正式版补丁包，确保 GitHub 与 Gitee 下载到的同一份 Windows 便携包都内置双更新源。
- 发布脚本会自动读取本机 `gitee` remote 注入国内镜像更新源，减少手工漏配导致的单源包。
- Gitee Release 上传前会检查 zip 内 `release-manifest.json`，缺少 GitHub 或 Gitee 任一更新源时拒绝发布。
- 发布包和包内主程序继续使用“阿谢姆水晶（Azem's Crystal）”品牌命名。

## 0.1.11

- 修复 Tauri 桌面端点击“官方详情”没有打开系统默认浏览器的问题。
- 桌面端接入 Tauri opener 插件；普通浏览器/Node 便携版继续使用原生新标签页打开官方详情。
- 桌面端权限补充 `opener:default`，并将 Rust 最低版本对齐到插件要求的 `1.77.2`。

## 0.1.10

- 修复 Windows PowerShell 5.1 解析自更新脚本失败，导致“下载完成但不覆盖、不重启”的问题。
- 自更新脚本改为带 UTF-8 BOM 写入，兼容中文安装路径和脚本中的中文错误信息。
- 启动覆盖脚本前先写入 `apply-update.log` 的准备记录，后续可区分“脚本未启动”和“覆盖阶段失败”。

## 0.1.9

- 修复当前版本高于所选下载节点最新 Release 时误显示“Release 对齐”的问题。
- 当国内镜像尚未同步主发布源时，更新状态改为黄色“节点待同步”，提示可切换节点或等待镜像发布。
- 更新状态判断新增版本关系单测，覆盖镜像落后、版本对齐和重大版本落后三类场景。

## 0.1.8

- 修复一键更新下载 Release zip 时可能出现 `更新包读取失败：error decoding response body` 的问题。
- 桌面版更新下载改用二进制下载专用客户端：更长超时、禁止压缩响应编码，并对临时网络错误自动重试。
- Node 便携版更新下载同步加入二进制响应头和重试，减少国内外下载节点链路波动造成的失败。
- GitHub 更新包下载优先复用 Windows 系统代理设置；国内镜像保持直接下载。
- 一键更新覆盖脚本增加本地日志，便于排查下载成功但没有自动安装/重启的问题。

## 0.1.7

- 优化“我的职业可进”：当招募未填写需求职业时，会根据 MT/ST、H1/H2、D1-D4 空缺位置智能判断所选职业是否可进。
- DPS 位置兜底保持宽松：近战、远敏、法系不会被默认站位规则过度过滤。
- “我的职业可选”按智能分类、防护、治疗、近战、远敏、法系顺序展示。
- “空缺位置”移动到“我的职业可进”上方，方便优先按位置快速筛选。
- Release 构建要求同时内置 GitHub 和国内镜像更新源；缺少国内镜像配置时发布构建会失败，避免再次产出单源包。

## 0.1.6

- 修复 Tauri 桌面便携包误加载 `http://127.0.0.1:5173` 开发地址的问题。
- 桌面便携构建继续补齐 VS / Windows SDK 环境，但最终统一走 `tauri build --no-bundle`，确保正式包加载内嵌前端资源。
- 重新发布桌面便携包，用户解压后可直接双击运行，不需要启动 Vite、Node 或本地 Express 服务。

## 0.1.5

- 新增便携包一键更新：用户确认后自动下载当前客户端对应的 zip，退出程序、覆盖当前解压目录并重启新版。
- 更新检查卡片优先显示“更新包”和“一键更新”按钮，不再把发布页跳转作为主要路径。
- 优化“拉取全量/刷新全量”主按钮视觉状态，禁用时改为浅色提示，避免看起来像黑色异常按钮。
- Gitee Release 附件识别兼容 `attach_files` / `download_url`，方便国内镜像节点返回可安装更新包。
- 修复普通 PowerShell 构建桌面便携包时 Cargo、VS C++ 与 Windows SDK 环境不稳定的问题。
- `npm run release:gitee` 上传 zip 时同步上传 `.sha256`，并对网络中断、5xx、429 等临时错误自动重试。

## 0.1.4

- 修复 Tauri 桌面便携版无法读取国内镜像更新源配置的问题。
- Tauri 桌面便携包的 `release-manifest.json` 现在会写入 `updateRepositories`。
- GitHub Release workflow 构建 Tauri 桌面便携包时同步注入国内镜像配置。
- 明确：上传已有 zip 到 Gitee 不会改写客户端内置更新源，发布镜像包前需要先带配置重新构建。

## 0.1.3

- 新增 Tauri 桌面便携包构建链，产物为解压后直接双击的桌面 EXE。
- 桌面版不再启动本地 Express 服务，也不需要用户安装 Node.js。
- 新增 Tauri 应用图标和桌面便携包中文说明。
- 前端版本状态支持显示“桌面版”，避免 Tauri 运行时误显示“开发模式”。
- 保留 NSIS 安装包目标；若 NSIS 下载超时，可先发布桌面便携包。

## 0.1.2

- Windows 便携包新增 `RisingStones-PartyFinder.exe` 一键启动入口。
- 用户解压 zip 后可直接双击 EXE 启动本地服务并自动打开浏览器。
- 保留 `start-windows.bat` 作为备用启动入口。
- 便携包构建脚本接入 Node.js SEA 和 `postject` 生成 EXE。
- 本地服务支持 `AUTO_OPEN_BROWSER=true` 自动打开本地页面。
- 补充 Tauri 桌面客户端原型源码和文档，为后续签名安装包与自动更新做准备。
- 补充代码签名准备说明。

## 0.1.1

- 公开源码和文档不再包含国内镜像真实仓库地址。
- 国内镜像发布源改为通过本机环境变量、未提交本地配置或便携包 manifest 注入。
- 更新检查不再由前端传递仓库路径，只显示“GitHub”和“国内镜像”节点。
- 前端首次进入时等待 GeoIP 节点推荐完成后自动检查更新。
- 未配置国内镜像时自动回退 GitHub，避免首次进入显示镜像未配置错误。
- 新增 `npm run release:gitee`，用于发布机本地创建/更新 Gitee Release 并上传便携包。
- 增加令牌泄露处理说明：已暴露令牌必须撤销并重新生成。

## 0.1.0

- 初始化本地网页工具：React + Vite + TypeScript 前端，Node/Express 本地代理。
- 支持按选定副本全量分页拉取石之家公开招募列表。
- 支持官方条件：副本类型/名称、大区、标签、队伍构成、位置、团队 A/B/C。
- 支持本地筛选：进度、攻略、时间、全局不包含关键词、我的职业可进、无重复职业、空缺位置。
- 支持结果卡片展开官方队伍详情、招募要求、攻略说明。
- 支持桌面端固定左侧筛选器和折叠/展开。
- 提供官方页面 Tampermonkey 手动响应助手脚本。
- 补齐 MIT 许可、中文 README、贡献指南、安全策略、发布包说明和 GitHub 工作流。
- 支持生成 Windows x64 开箱即用便携包。
