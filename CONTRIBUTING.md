# 贡献指南

感谢你愿意改进这个工具。这个项目的目标是提供一个安全、透明、可审计的本地筛选器，而不是账号自动化工具。

## 开发环境

需要 Node.js 20 或更高版本。

```powershell
npm ci
npm run dev
```

常用验证：

```powershell
npm test
npm run build
```

发布包验证：

```powershell
npm run build:portable
```

## 提交前检查

提交 Pull Request 前请确认：

- 新功能或行为变化已经更新中文文档。
- 修改筛选语义时补充或更新单元测试。
- 没有提交账号、Cookie、Token、联系信息或本地 `.env` 文件。
- 没有加入自动响应、批量响应或绕过官方登录的能力。
- 对官方接口的请求仍然保留分页上限、请求间隔、取消和错误提示。

## 推荐分支和提交

- 分支名建议使用 `feature/<简短说明>`、`fix/<简短说明>` 或 `docs/<简短说明>`。
- Commit message 建议使用中文，说明“改了什么”和“为什么改”。

## Issue 建议

提交问题时请尽量包含：

- 操作系统和浏览器。
- 使用的是源码运行还是 Release 便携包。
- 复现步骤。
- 期望结果和实际结果。
- 控制台错误或截图，注意不要包含账号、Cookie、Token、联系方式。

## 设计原则

- 本地工具只做公开招募数据读取和本地筛选。
- 账号相关操作必须留在官方登录态和用户手动确认路径中。
- 优先小步迭代，保持筛选逻辑可测试。
- 面向普通玩家的文档优先使用中文。
